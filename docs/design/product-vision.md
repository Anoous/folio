# Folio 产品愿景 — "你的第二个大脑"

> 创建日期：2026-03-22
> 灵感来源：以乔布斯式产品思维重新审视 Folio 的定位与差异化

---

## 核心洞察：收藏不是目的

人们收藏文章的真正原因不是"保存"，而是**害怕忘记**。

"这篇文章说得太好了，我以后肯定用得上。"

然后呢？99% 的人再也没打开过。不是因为懒，而是**找不到那个"以后"的时刻**。

收藏工具解决的是错误的问题。它们让"存"变得更容易，但"存"从来不是瓶颈——**"用"才是。**

---

## 定位转变

```
旧定位："分享链接，知识留住。"（描述动作，工具属性）
新定位："你的第二个大脑。它记得你读过的一切。"（描述价值，资产属性）
```

工具用完就走，资产越积越值钱。Folio 的价值应该随数据复利增长——存得越多，问答越准，越离不开。

---

## 杀手场景

产品只需要解决一个场景：

> **"我之前看过一篇……讲什么来着……"**

每个人每周都会遇到这个时刻。写方案时、和同事聊天时、刷到相关话题时——你知道你读过，但想不起来细节，也找不到。

### 演示时刻

用户存了 200 篇文章，大部分存完就忘了。某天准备一个关于 AI 的演讲，打开 Folio，在搜索框输入：

**"训练大模型最反直觉的发现是什么"**

0.5 秒后：

```
基于你收藏的 7 篇相关文章，核心发现是：

"数据质量比数据量更重要。1T 高质量 token
 胜过 10T 噪声数据。"
 — 来自《Scaling Laws 的启示》，3 月 2 日收藏

3 篇文章持相同观点，但《Chinchilla 论文解读》
提出了不同角度：模型规模和数据量存在最优比例……

→ 查看全部 7 篇相关文章
```

用户没有记住任何一篇文章的标题，甚至不记得存过它。但 Folio 记得。

## 减法设计：砍掉什么

| 砍掉 / 降级 | 为什么 |
|-------------|--------|
| 9 个分类标签栏 | 用户不应该做机器的工作。iPhone 从不让你手动给照片分类 |
| 标签管理页面 | 标签是给机器看的索引，不是给人看的 UI |
| "按分类浏览" | 没有人因为"想看科技类"打开 Folio，他们带着具体问题来 |
| 复杂设置 | 选项越多，产品越弱 |

---

## 三个界面，三种状态

### 1. Home — 你的知识流

不是按时间排列的列表。是一条有呼吸感的信息流：

- 最近保存的文章（带洞察级摘要）
- 偶尔穿插"知识回响"：*"你 2 周前存的这篇，和昨天存的这篇观点相反——"*
- 没有分类栏，没有标签云，没有筛选器

### 2. Search — 你的第二个大脑

不是关键词搜索。是对话式检索：

- 输入自然语言："那个用咖啡渣做建材的创业公司"
- 即使原文是英文、你记错了细节，也能找到
- 搜索结果不是 10 条链接，而是**一个答案 + 来源**

### 3. Reader — 沉浸阅读

打开文章时，底部安静地显示：*"你的收藏中有 3 篇相关文章"*

不打扰，但在你需要时，知识之间的连接已经建好了。

## 情感曲线：越用越离不开

```
第 1 天（存第 1 篇）：  "摘要写得不错，比我总结的好"
第 7 天（存第 10 篇）： "搜了一下，居然找到了那篇想不起来的文章"
第 30 天（存第 50 篇）："它告诉我两篇文章观点矛盾，我自己都没注意到"
第 90 天（存第 200 篇）："我开始依赖它写方案了——问它比翻收藏快 10 倍"
```

价值随数据复利增长。这就是护城河。

## 差异化优先级

| 优先级 | 功能 | 预估工作量 | 差异化价值 |
|--------|------|-----------|-----------|
| 1 | F10 洞察级摘要（改 prompt） | 1-2 天 | 让"存"的瞬间就有惊喜 |
| 2 | 向量嵌入基础设施 | 核心工程 | 一切高级功能的地基 |
| 3 | F12 自然语言问答（RAG） | 大工程 | 杀手功能，用户留下来的唯一理由 |
| 4 | 语义搜索 + 相关收藏 | 复用 F12 基础设施 | 自动解锁 |

## 竞品对比

| | Pocket / Instapaper | Raindrop / Cubox | **Folio（愿景）** |
|---|---|---|---|
| 核心能力 | 稍后阅读 | 书签 + 手动分类 | **个人知识库 + AI 问答** |
| 价值随时间 | 不变（越存越乱） | 缓慢增长（靠人工整理） | **复利增长（AI 自动连接）** |
| 找回内容 | 关键词搜索 | 关键词 + 标签筛选 | **自然语言问答** |
| 差异化 | 离线阅读体验 | 视觉美观 | **"它记得你读过的一切"** |

## 三大引力场：让用户每天回来

当前 Folio 是被动的——用户存东西进去，需要时来找。伟大的产品是主动的——它给你理由每天回来。

### 引力场 1：知识入口的革命 — 让知识无摩擦地流入

当前只有一个入口：分享链接。但知识不只存在于链接里。

#### A. 截图即收藏

微信群里的精彩发言、Kindle 上的一页好书、朋友圈的图文——这些都没有"链接"可以分享。

**方案**：截图 → Folio 自动 OCR（iOS Vision framework，端侧免费） → AI 提取核心内容 → 入库

**为什么重要**：让 Folio 从"文章收藏工具"变成"万物收藏工具"。截图是最通用的输入方式。

#### B. 语音捕捉

走在路上、开车时——灵感来了，掏不出手机打字。

**方案**：长按 Widget / 快捷指令 → 说一句话 → AI 转文字 + 结构化 → 入库

> "记住：今天播客里说，数据飞轮比算法更重要"

**为什么重要**：知识的最大损耗不是"没保存链接"，而是想法一闪而过就忘了。语音是捕捉瞬间想法唯一不打断心流的方式。

#### C. 轻量笔记输入

不是笔记编辑器，而是最轻量的知识输入。一个输入框，写一句话或一段话，AI 自动分类、打标签，和文章收藏平等地存在于知识库中。

> "张三推荐了 Heptabase，说比 Notion 更适合做知识管理"

当你将来搜索"知识管理工具"时，这条知识碎片会和相关文章一起出现。

---

### 引力场 2：知识的主动唤醒 — 让沉睡的知识找到你

这是让用户每天打开 Folio 的关键。

#### D. 每日知识回响（Daily Echo）

每天早上，一张精心设计的卡片：

```
┌─────────────────────────────────────┐
│  ✦ 今日回响                          │
│                                      │
│  你 3 周前收藏了《为什么 90% 的        │
│  AI 项目失败》，其中一个核心观点：      │
│                                      │
│  "失败的根因不是技术，而是问题          │
│   定义错误——团队在用 AI 解决          │
│   不需要 AI 的问题。"                 │
│                                      │
│  有趣的是，你昨天存的《如何评估 AI      │
│  项目可行性》给出了互补视角……          │
│                                      │
│  → 查看这两篇的关联                   │
└─────────────────────────────────────┘
```

**设计原则**：
- 不是随机推送，是 AI 选择此刻最值得回顾的洞察
- 优先选择：与近期收藏有关联的旧文章、被遗忘的高质量文章、观点冲突的文章对
- 每天只推 1 条，克制。少即是多

**为什么重要**：Readwise 证明了"每日回顾"能创造强烈的用户习惯。但 Readwise 回顾的是高亮标注，Folio 回顾的是 AI 提炼的洞察 + 跨文章关联——高一个量级。

#### E. 锁屏 Widget

iOS 锁屏上，一行你收藏过的洞察。每次亮屏看一眼，潜移默化。

```
✦ "好的 API 设计像好的笑话——不需要解释" — 你 2 周前收藏
```

这不是功能，这是品牌心智的占领。用户每天看 100 次锁屏，每次都被提醒：我的知识库在增长。

#### F. 知识图谱可视化

不是复杂的 graph view，而是简单的月度知识地图：

```
这个月你的知识版图

     AI ━━━━━━━━━━ 12 篇
      ┃
    产品设计 ━━━━━━ 8 篇
      ┃
    创业 ━━━━━━━━━ 5 篇
           ┃
         投资 ━━━━ 3 篇

你的关注重心正在从「AI 技术」转向「AI 产品化」
```

**为什么重要**：人们喜欢看到自己的成长轨迹。收藏行为有了意义感——你不是在囤链接，你在建设知识版图。

---

### 引力场 3：知识的输出 — 让收藏变成你的武器

让 Folio 从"好用"变成"不可替代"的关键。

#### G. 知识简报导出

> "我要准备一个关于 AI Agent 的分享，把我相关的收藏整理一下"

Folio 基于你收藏的文章，生成结构化简报：

```markdown
# AI Agent — 基于你的 9 篇收藏

## 核心共识
- Agent 的关键不是模型能力，而是工具调用的可靠性（5 篇提及）

## 争议观点
- 《XX》认为多 Agent 协作是方向，但《YY》认为单 Agent + 好工具更实际

## 关键数据
- "2025 年 Agent 市场规模预计 $XX 亿" — 来自《ZZ》

## 原文链接
1. ...
```

**为什么重要**：这是"第二个大脑"承诺的终极兑现——收藏不再是沉睡的链接，而是随时可以调用的研究助手。

#### H. 阅读时高亮标注

在 Reader 中长按文字 → 高亮。高亮内容成为知识库的一等公民：

- 可独立搜索："我标注过什么关于用户留存的内容？"
- 在知识简报中优先引用
- 在 Daily Echo 中被回顾

**为什么重要**：高亮是用户告诉 Folio"这句话对我重要"的方式。没有高亮，AI 只能猜；有了高亮，AI 知道你在乎什么。

| 功能 | 类型 | 创造的价值 | 建议优先级 |
|------|------|-----------|-----------|

---

## 终极产品公式

```
Folio = 无摩擦输入（链接 + 截图 + 语音 + 笔记）
      + AI 深度理解（洞察摘要 + 自动关联）
      + 主动唤醒（每日回响 + Widget）
      + 知识问答（RAG）
      + 输出闭环（知识简报 + 高亮）
```

**一句话：知识流进来，洞察流出去，中间是 AI。**
