# RAG 问答 — Design Spec

> 日期：2026-03-23
> 状态：Approved
> 范围：P1.3 — RAG 问答系统（方案 C：上下文填充）

---

## 概述

用户在搜索页输入长问句，后端加载用户全部文章摘要塞进 LLM 上下文，LLM 综合回答并标注来源文章。无向量数据库、无嵌入管线，一次 LLM 调用搞定。

**核心洞察：** 个人知识库典型规模 50-500 篇文章，500 篇 × ~100 tokens ≈ 50K tokens，完全装进 DeepSeek 64K 上下文窗口。LLM 看到全部文章比任何检索算法都准。

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| RAG 方案 | 上下文填充（方案 C） | 零基础设施、最简单、LLM 直接读全部摘要最准 |
| 超过 500 篇 | FTS 预筛选 Top-50 再填充 | 降级策略，保证 context 不溢出 |
| 响应模式 | 同步返回（非 SSE） | 500 篇摘要 LLM 几秒内回答，不需要流式 |
| 搜索分流 | 短查询→FTS，长问句→RAG | 输入 >10 字 + 疑问词/问号 触发 RAG |
| 对话支持 | 支持跟进提问（conversation_id） | 复用 rag_conversations + rag_messages 表 |

## 架构

```
用户输入长问句 → POST /api/v1/rag/query
                    ↓
后端加载全部文章 title + summary（≤500 篇直接塞上下文）
超过 500 篇 → pg_trgm 搜索 Top-50 再塞
                    ↓
DeepSeek LLM（64K context, temperature 0.3, max_tokens 2048）
Prompt: 系统指令 + 全部文章摘要 + 用户问题
                    ↓
解析回答 + 引用的文章 ID + 生成跟进建议
                    ↓
写入 rag_conversations + rag_messages
                    ↓
返回：answer + sources[] + followup_suggestions[]
```

## 后端设计

### 1. API 端点

#### POST /api/v1/rag/query

**请求：**
```json
{
  "question": "关于用户留存有哪些方法？",
  "conversation_id": null
}
```

**响应：**
```json
{
  "answer": "基于你的收藏，留存的本质是习惯设计，不是功能堆砌。[1] 提出了触发→行动→奖赏→投入的模型...",
  "sources": [
    {
      "article_id": "uuid",
      "title": "用户留存的本质：习惯回路与变量奖赏",
      "site_name": "少数派",
      "summary": "留存不是功能问题，是行为设计问题...",
      "relevance": 0.95
    }
  ],
  "source_count": 3,
  "followup_suggestions": [
    "哪些文章提到了飞轮效应？",
    "留存和增长的关系是什么？"
  ],
  "conversation_id": "uuid"
}
```

**逻辑：**
1. 验证 auth + 检查 Free 配额（`rag_count_this_month`，Free 限 5 次/月）
2. 若有 `conversation_id`：加载历史消息（最近 10 条）作为对话上下文
3. 加载用户全部文章（id, title, summary, site_name）
4. 若文章数 ≤ 500：全部填充到 prompt
5. 若文章数 > 500：用 pg_trgm 搜索 `question` 关键词，取 Top-50 填充
6. 构建 prompt（见下方）
7. 调用 DeepSeek Chat API（temperature 0.3, max_tokens 2048, json_object response）
8. 解析 JSON 响应：answer, cited_article_indices[], followup_suggestions[]
9. 将 indices 映射为实际 article_id
10. 写入 rag_conversations（新建或追加）+ rag_messages
11. 递增 Free 用户 `rag_count_this_month`
12. 返回响应

**权限：** auth
**配额：** Free 5 次/月，Pro 无限

### 2. AI Prompt

**系统 prompt：**
```
你是用户的个人知识助手。以下是用户收藏的文章摘要列表。
基于且仅基于这些文章回答用户的问题。

输出 JSON 格式（不要 markdown 代码块）：
{
  "answer": "回答正文，在引用处用 [1] [2] 等标注对应文章编号",
  "cited_indices": [1, 3, 5],
  "followup_suggestions": ["建议追问1", "建议追问2"]
}

规则：
1. 只基于用户的收藏回答，不编造内容
2. 引用标注 [N] 必须对应下方文章列表的编号
3. 回答风格：简洁、有洞察力、直击核心
4. 给出 2 个建议的跟进问题
5. 如果收藏中没有相关内容，answer 写 "你的收藏中没有找到与此相关的内容。"，cited_indices 为空
```

**用户 prompt：**
```
用户收藏（共 {count} 篇）：

[1] 《{title}》({site_name}): {summary}
[2] 《{title}》({site_name}): {summary}
...

{对话历史（如果有 conversation_id）：}
用户：{之前的问题}
助手：{之前的回答}
...

用户问题：{question}
```

### 3. 配额重置

和 Echo 类似：
- `rag_count_this_month` + `rag_month_reset_at`
- GET /rag/query 时检查：如果 `rag_month_reset_at` < 本月 1 日，重置为 0

## iOS 设计

### 1. 搜索分流逻辑

在搜索内容区域（HomeView 的 `searchContent`），根据输入判断：

```swift
func isRAGQuery(_ text: String) -> Bool {
    let trimmed = text.trimmingCharacters(in: .whitespacesAndNewlines)
    guard trimmed.count > 10 else { return false }
    let questionIndicators = ["？", "?", "什么", "哪些", "如何", "为什么", "怎么", "怎样", "是否", "有没有", "能不能", "多少"]
    return questionIndicators.contains { trimmed.contains($0) }
}
```

- `isRAGQuery == false` → 现有 FTS 搜索
- `isRAGQuery == true` → 显示 RAG 回答 UI + 触发 POST /rag/query

### 2. RAG 回答 UI（匹配原型 03）

**文件：** `ios/Folio/Presentation/Search/RAGAnswerView.swift`

**布局：**
- "✦ 基于 N 篇收藏" 标签（11px, accent, 和 Echo 标签同风格）
- 回答正文（衬线体 17px, line-height 1.65）
  - 行内引注 [1] [2] 显示为 accent 色上标
  - 点击引注 → 展开对应来源卡片
- 来源文章卡片列表：
  - 标题 + 来源 + 摘要摘录
  - 点击 → 跳转到 Reader
- 建议追问（2 个按钮，pill 样式）
  - 点击 → 填入搜索框，重新查询
- 加载中状态："✦ 正在思考..." 动画

### 3. DTO + APIClient

```swift
struct RAGQueryRequest: Codable {
    let question: String
    let conversationId: String?
}

struct RAGQueryResponse: Codable {
    let answer: String
    let sources: [RAGSource]
    let sourceCount: Int
    let followupSuggestions: [String]
    let conversationId: String
}

struct RAGSource: Codable {
    let articleId: String
    let title: String
    let siteName: String?
    let summary: String?
    let relevance: Double
}

// APIClient method:
func ragQuery(question: String, conversationId: String? = nil) async throws -> RAGQueryResponse
```

- RAG 回答出现在搜索结果位置

## 不做

- SSE 流式响应（同步足够）
- 向量嵌入
- 独立 RAG 页面（在搜索页内完成）
- 离线 RAG
- RAG 对话历史页面
- 保存 RAG 回答为笔记

## 对应原型 03 元素

| 原型元素 | 本设计对应 |
|---------|-----------|
| "✦ Folio 问答" 标签 | RAGAnswerView 顶部标签 |
