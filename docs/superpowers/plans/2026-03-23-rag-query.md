# RAG Query Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build RAG Q&A — user asks a natural language question, backend loads all article summaries into LLM context, LLM synthesizes an answer with source citations, displayed inline in the search page with follow-up conversation support.

**Architecture:** Context stuffing approach — no vector DB. Backend loads all user article summaries (≤500 direct, >500 FTS pre-filter), constructs a prompt, calls DeepSeek synchronously, returns structured JSON (answer + sources + follow-ups). iOS: search page splits short queries (FTS) vs long questions (RAG), RAGAnswerView renders answer with inline citations, source cards, and follow-up thread.

**Tech Stack:** Go 1.24 / chi v5 / pgx v5 / DeepSeek API | Swift 5.9 / SwiftUI / iOS 17.0

**Spec:** `docs/superpowers/specs/2026-03-23-rag-query.md`

## Task 2: Backend — RAG Service (Prompt + LLM + Token Budget)

**Files:**
- Create: `server/internal/service/rag.go`

- [ ] **Step 1: Create service/rag.go**

```go
type RAGService struct {
    ragRepo     *repository.RAGRepo
    userRepo    *repository.UserRepo
    aiClient    *client.DeepSeekAnalyzer
}

func NewRAGService(ragRepo *repository.RAGRepo, userRepo *repository.UserRepo, aiClient *client.DeepSeekAnalyzer) *RAGService
```

**Main method: `Query(ctx, userID, question, conversationID string) (*domain.RAGResponse, error)`**

Logic:
1. Check quota: load rag_count_this_month, if month reset needed → reset. Free limit = 5, return error if exceeded.
2. If article count == 0 → return answer "先收藏一些文章再来提问吧。" with empty sources.
3. Load all article summaries.
4. **Token budget**: truncate each summary to ~200 Chinese characters (~150 tokens). Count cumulative. If >500 articles or cumulative >50K tokens → fall back to `SearchArticleSummaries(query, 50)`.
5. If conversationID non-empty → load last 10 messages.
6. **Sanitize question**: remove ``` and role markers (same as ai.go sanitization).
7. Build prompt (system + user).
8. Call DeepSeek: model=deepseek-chat, temperature=0.3, max_tokens=2048, response_format=json_object, **timeout=30s**.
9. Parse JSON response → answer, cited_indices, followup_suggestions.
10. Map cited_indices to actual article IDs. Filter invalid indices.
11. If JSON parse fails → return fallback answer "抱歉，回答生成失败，请重试。"
12. Create/update conversation + add messages (user question + assistant answer).
13. Increment quota.
14. Return RAGResponse.

**Prompt construction** (see spec for exact prompts):
- System: "你是用户的个人知识助手...输出 JSON..."
- User: "[1] 《title》(source, date): summary\n..." + conversation history + question

**Helper: `truncateSummary(s string, maxRunes int) string`** — truncate to 200 runes with "…" suffix.

struct RAGQueryRequest: Codable {

    let sources: [RAGSource]
    let sourceCount: Int

    let articleId: String
    let title: String
    let siteName: String?
    let summary: String?
    let createdAt: Date
    let relevance: Double

// MARK: - RAG API

func ragQuery(question: String, conversationId: String? = nil) async throws -> RAGQueryResponse {
    let body = RAGQueryRequest(question: question, conversationId: conversationId)
    return try await request(method: "POST", path: "/api/v1/rag/query", body: body)
}

- [ ] **Step 1: Add RAG state to HomeViewModel**

```swift
// RAG state
var ragResponse: RAGQueryResponse?
var ragIsLoading = false
var ragError: String?
var ragConversationId: String?

// Follow-up messages for inline thread
var ragThread: [(question: String, response: RAGQueryResponse)] = []

func isRAGQuery(_ text: String) -> Bool {
    let trimmed = text.trimmingCharacters(in: .whitespacesAndNewlines)
    guard trimmed.count > 10 else { return false }
    let indicators = ["？", "?", "什么", "哪些", "如何", "为什么", "怎么", "怎样", "是否", "有没有", "能不能", "多少"]
    return indicators.contains { trimmed.contains($0) }
}

// 1s debounced RAG query
private var ragDebounceTask: Task<Void, Never>?

func submitRAGQuery(_ question: String) {
    ragDebounceTask?.cancel()
    ragDebounceTask = Task {
        try? await Task.sleep(for: .seconds(1))
        guard !Task.isCancelled else { return }
        await executeRAGQuery(question)
    }
}

@MainActor
private func executeRAGQuery(_ question: String) async {
    ragIsLoading = true
    ragError = nil
    do {
        let response = try await apiClient.ragQuery(
            question: question,
            conversationId: ragConversationId
        )
        ragResponse = response
        ragConversationId = response.conversationId
    } catch let error as APIError {
        if case .quotaExceeded = error {
            ragError = "quota"
        } else {
            ragError = "error"
        }
    } catch {
        ragError = "error"
    }
    ragIsLoading = false
}

func submitFollowup(_ question: String) {
    // Append current response to thread before making new query
    if let current = ragResponse {
        ragThread.append((question: ragResponse?.answer ?? "", response: current))
    }
    submitRAGQuery(question)
}

func clearRAG() {
    ragResponse = nil
    ragConversationId = nil
    ragThread = []
    ragError = nil
}
```

## Risk Notes

- **Task 2 (RAG Service)** is the most complex — prompt construction, token counting, and response parsing all happen here. The token budget logic should be conservative (truncate aggressively rather than risk context overflow).
- **Task 5 (RAGAnswerView)** requires parsing the answer text to extract citation superscripts and render them as tappable elements. This is non-trivial in SwiftUI — consider using `AttributedString` or splitting the text by citation markers.
- **Task 6 (Search Integration)** modifies HomeView which is already large. Be careful with the search text onChange handler — the RAG debounce and FTS search need to coexist without conflicting.
